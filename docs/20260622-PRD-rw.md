# PRD：R&W 综合练习游戏模块

**版本**：v0.2（含对话设计讨论完整记录）  
**日期**：2026-06-22  
**产品背景**：SAT Game —— 为 Isabella（Grade 9，加拿大）打造的每日游戏化 SAT 备考 APP

---

## 一、模块定位

**目标**：针对 SAT R&W（Reading & Writing）四大考点的专项游戏化练习，含完整学习引擎。

**核心设计原则**：

1. **玩法本身就是练习**，不是选择题换皮
2. **Generation > Recognition**：让用户主动生成答案，而非从选项中识别
3. **掌握度记在考点维度**，不记在单个题目（同一考点换句话出题）
4. **两种重来**：即时迁移（换句，同规则）+ 间隔重现（换语境，数天后）
5. **英英优先**：词义解释默认英文，按需中文翻译
6. **无暗黑设计**：不设生命条/体力条，错误是学习的一部分，不惩罚

## 三、五个游戏设计

### 3.1 Closer（语义测距猜词）

**考点**：词在语境中的含义（Words in Context）

**玩法机制**：

```
[一个句子，其中目标词被 [___] 替换]
[提示：给出若干候选词，按语义距离排列]
用户滑动"语义尺"，选择最符合语境的词
系统判断：选中词与正确答案的语义距离
```

**核心设计**：
- 语义距离由 LLM（Claude）实时计算，不是预设答案
- 用户可以多次调整滑块，每次提交都有反馈
- 正确答案需要同时考虑：词义本身 + 句子语境 + 词性匹配
- 答对后显示完整词义解释卡

**技术依赖**：
- `/api/semantic-distance` 接口，调用 Anthropic Claude API
- 需要配置 `ANTHROPIC_API_KEY`

**难度分级**：
- 简单：候选词差异大，语境明显
- 中等：候选词语义接近，需要细读语境
- 困难：近义词中细微差异决定正确答案

---

### 3.2 Stronger（近义词强弱阶梯，2048 式）

**考点**：近义词精确度（Synonym Precision）

**玩法机制**：

```
[棋盘上有若干词块]
相邻两块词义相近 → 合并 → 生成更强/更弱的同义词
目标：合成到目标强度的词（如从 "content" 合成到 "ecstatic"）
```

**词链示例**（从弱到强）：
```
content → pleased → happy → joyful → elated → ecstatic
calm → peaceful → serene → tranquil
```

**核心设计**：
- 每条词链代表一个语义维度（情绪强度、正式程度、语气强弱等）
- 合并时需要用户确认两词的语义关系
- 错误合并（语义不连续）= 失败，触发即时迁移
- Word 模型中 `intensity` 字段（1-5）支撑词链强弱

**数据模型支撑**：
```prisma
// Word.intensity: Int  // 1-5，表示词义强弱
// Word.synonyms: String[]  // 关联同义词 id
// Word.tone: String  // positive/negative/neutral
```

---

### 3.3 Families（词义归类，三消式）

**考点**：词义归类（Word Meaning Classification）

**玩法机制**：

```
[棋盘上有词块，带颜色分类标记]
消除前：必须回答该词属于哪个"家族"（分类）
回答正确 → 标记为该分类颜色 → 可与同类词相邻消除
回答错误 → 进入即时迁移（换句，同分类）
```

**分类示例**：
- 描述「抵抗/坚持」的词：resilient, tenacious, steadfast, persistent
- 描述「批评/质疑」的词：scrutinize, critique, rebuke, censure
- 描述「支持/辩护」的词：advocate, champion, endorse, espouse

**核心设计**：
- 消除前的分类问答是核心练习
- 同一分类词义细微差异在消除后展示
- 三消动画作为正确反馈的奖励

---

### 3.4 Gate Run（跑酷双门，限时选择）

**考点**：标点规则 + 过渡词使用（Punctuation & Transition Words）

**玩法机制**：

```
[角色向右跑动，迎面而来两扇门]
每扇门上有一个选项：
  - 一个正确的标点/连接词
  - 一个错误的标点/连接词
用户点击正确的门让角色跑过
答错 → 角色跌倒 → 显示解析 → 继续下一关
```

**题目示例**：
```
句子：The experiment failed; ___ the team did not give up.
门A：however  ✓
门B：therefore ✗
```

**考点覆盖**：
- 分号、冒号、逗号的使用场景
- 因果类过渡词（therefore, thus, consequently）
- 转折类过渡词（however, nevertheless, despite）
- 递进类过渡词（furthermore, moreover, additionally）
- 举例类过渡词（for instance, specifically, namely）

**核心设计**：
- 限时（每扇门 15 秒倒计时）增加紧张感
- 速度模式：连续答对加速，答错减速（视觉反馈）
- 答对积累连续数，显示在屏幕顶部

---

### 3.5 Pair Up（连连看，词义配对）

**考点**：词义配对（基础词汇 + 近义反义识别）

**玩法机制**：

```
[两列，左列词，右列选项（释义/近义词/反义词）]
用户连线：将词与对应释义/近反义词匹配
全部正确 → 过关
错误 → 连线消失，记录错误
```

**三种题型**：
1. 词 ↔ 英文释义（词义理解）
2. 词 ↔ 近义词（近义识别）
3. 词 ↔ 反义词（反义识别）

**核心设计**：
- 每局 6-8 对词，难度适中
- 错误词对自动加入生词本
- 支持「混合模式」（三种题型混合出现）

## 四、学习引擎（SRS 系统）

### 4.1 掌握度模型

掌握度**不记在具体题目上，记在考点维度（skill/rule）上**：

```
考点 = {
  skill: "word_in_context" | "synonym_precision" | "punctuation" | "transition_words" | ...
  rule: "contrast_conjunction" | "causal_conjunction" | "colons_before_list" | ...
}
```

同一个考点可以用不同的句子/词来出题，考的是「规则掌握」而非「题目记忆」。

### 4.2 掌握度升降规则

```
连续答对 3 次 → 该考点"毕业"（masteryScore = 1.0，nextReview = 7天后）
答错 1 次 → 打回最短间隔（nextReview = 1天后，masteryScore 降级）
```

### 4.3 SRS 间隔曲线（艾宾浩斯遗忘曲线）

| 连续答对次数 | 下次复习间隔 |
|-------------|-------------|
| 第 1 次 | 1 天 |
| 第 2 次 | 3 天 |
| 第 3 次（毕业） | 7 天 |
| 毕业后稳固 | 14 天 → 30 天 |

答错重置为 1 天，不论之前积累了多少次。

### 4.4 两种"重来"

**即时迁移（Immediate Transfer）**：
- 触发条件：答错后
- 形式：换一个句子，但考同一个规则/考点
- 目的：趁热打铁，不让错误停留在错误上

**间隔重现（Spaced Return）**：
- 触发条件：SRS 计时到期
- 形式：换一个语境（不同句子、不同词），考同一个考点
- 目的：在遗忘发生前强化记忆

### 4.5 生词本（VocabNote）

- 游戏中答错的词，自动加入生词本
- `addedFrom` 记录来源（closer/stronger/families/gaterun/pairup）
- `masteryLevel`：0=新词、1=学习中、2=已掌握
- `nextReview` 到期时，该词织回游戏作为出题材料

## 五、词义解释卡

所有 R&W 游戏答题后，显示词义卡：

```
[词]（词性）
--------------------
[英文释义]

Example: [英文例句]

[译成中文] ← 按钮，点击展开

（中文释义）
（例句中文翻译）
```

设计原则：
- 默认英英，减少依赖翻译的惰性
- 中文翻译是辅助工具，不是主要学习路径
- 按钮文字：「译成中文 / Show Chinese」

## 六、内容生产管线

### 6.1 离线批量生产（主要路径）

```
LLM 批量生成 → 人工审核 → 写入数据库（stable ID）→ 游戏调用
```

生成内容包括：
- 词义、例句（英英）
- Connections 分组数据
- Gate Run 题目（句子 + 正确/错误标点连接词）
- Closer 题目（含挖空句子 + 候选词）
- Stronger 词链数据
- Families 分类数据

### 6.2 动态 LLM 调用（极少数场景）

只有以下两种情况需要实时调用 Claude API：
1. **Closer 游戏语义距离判断**：用户选词后，判断与正确答案的语义距离
2. **「用另一种方式解释」按钮**：用户要求换个角度理解词义

所有其他游戏内容均为预生成静态内容。

### 6.3 内容审核标准

- 词义准确（符合 SAT 考试级别）
- 例句自然（非机器感强的模板化句子）
- 难度分级合理（1=基础，2=中等，3=难）
- 无文化偏见内容

## 七、数据模型

```prisma
model Word {
  id            String   @id @default(cuid())
  word          String   @unique
  partOfSpeech  String   // noun/verb/adj/adv
  definitionEn  String
  definitionCn  String
  exampleEn     String
  tone          String   // positive/negative/neutral
  intensity     Int      // 1-5，用于 Stronger 游戏词链
  difficulty    Int      // 1-3，对应 SAT 难度
  category      String   // vocabulary/grammar/transition
  synonyms      String[] // 关联同义词 id
  antonyms      String[] // 关联反义词 id
  isActive      Boolean  @default(true)
  createdAt     DateTime @default(now())
}

model UserProgress {
  id           String    @id @default(cuid())
  wordId       String?
  mathId       String?
  gameType     String    // closer/stronger/families/gaterun/pairup
  skill        String    // 考点域
  result       String    // correct/incorrect
  attempts     Int       @default(1)
  masteryScore Float     @default(0)
  nextReview   DateTime?
  createdAt    DateTime  @default(now())
}

model VocabNote {
  id           String    @id @default(cuid())
  wordId       String
  addedFrom    String    // 来源游戏
  masteryLevel Int       @default(0)  // 0=新 1=学习中 2=掌握
  nextReview   DateTime?
  reviewCount  Int       @default(0)
  createdAt    DateTime  @default(now())
}
```
