# PRD：数学读题模块（Problem Dissector）

**版本**：v1.0  
**日期**：2026-06-22  
**产品背景**：SAT Game —— 为 Isabella（Grade 9，加拿大）打造的每日游戏化 SAT 备考 APP

---

## 一、模块定位

**核心定位**：练读题与解题规划，**不练计算**。

SAT 数学失分的常见原因不是不会算，而是：
1. 不知道题在问什么（读题不准）
2. 不知道用什么方法（题型识别弱）
3. 不知道解题步骤顺序（逻辑混乱）

Problem Dissector 专门针对这三个痛点，帮助 Isabella 建立系统的解题思维框架。

---

## 二、拆题三步流程

对同一道题，分三步练习，每步独立判断：

### 步骤一：在问什么？

```
[显示完整题目（隐藏答案）]
↓
[多个选项：题目真正在求哪个量？]
用户选择 → 系统判断对错 → 显示解析
```

**示例**：
- 题目：一个矩形面积为 48，长比宽多 2，求宽是多少？
- 错误答案：矩形的面积
- 正确答案：矩形的宽

考点：区分「已知量」和「待求量」，识别题目真正的问题。

### 步骤二：用什么方法？

```
[显示步骤一的正确答案]
↓
[多个选项：用哪种解题方法/公式？]
用户选择 → 系统判断对错 → 显示解析
```

**题型分类**（`archetype` 字段）：

| archetype | 中文 | 代表题型 |
|-----------|------|----------|
| linear | 线性方程 | 一元一次方程、直线斜率 |
| quadratic | 二次方程 | 配方、因式分解、求根公式 |
| percentage | 百分比 | 增减率、折扣、税率 |
| ratio | 比例 | 直接比例、正反比 |
| exponential | 指数 | 指数增长/衰减 |
| geometry | 几何 | 面积、体积、角度、三角形 |

**方法选项示例**：
- 线性方程组（联立求解）
- 二次方程（因式分解）
- 百分比公式
- 面积公式

### 步骤三：转几个弯？

```
[显示前两步正确答案]
↓
[乱序的解题步骤卡片]
用户拖动排序，按正确顺序排列
提交 → 系统判断顺序对错 → 显示完整算式
```

**示例步骤卡片**（需排序）：
1. 设宽为 x，则长为 x+2
2. 建立方程：x(x+2) = 48
3. 展开：x² + 2x - 48 = 0
4. 因式分解：(x+8)(x-6) = 0
5. 解方程：x = 6（舍负）
6. 答：宽为 6

**`turns` 字段**记录解题弯数（步骤数），用于难度参考。

## 三、功能特性

### 3.1 中英文切换

- 题目默认显示英文（SAT 原题语言）
- 一键切换为中文翻译版本
- 两种语言下的选项和解析同步翻译
- 目的：帮助理解题意，不依赖翻译（中文是脚手架，目标是看懂英文）

### 3.2 揭晓完整算式

- 步骤排序完成后（无论对错），显示完整解题过程
- 格式：逐步算式 + 中文解析
- 区分「解题方法选择」和「计算执行」（本模块不考后者）

### 3.3 难度标记

- 题目难度 1-3 级
- `turns`（弯数）作为辅助难度指标：弯数越多，解题越复杂
- 首页按难度筛选入口

## 四、数据模型

```prisma
model MathProblem {
  id          String   @id @default(cuid())
  textEn      String   // 题目正文（英文）
  textCn      String   // 题目正文（中文）
  questionEn  String   // 问句（英文）
  questionCn  String   // 问句（中文）
  archetype   String   // linear/quadratic/percentage/ratio/exponential/geometry
  turns       Int      // 解题弯数（步骤数）
  options     Json     // 三步的选项列表
  answer      Json     // 三步的正确答案
  explanation Json     // 完整步骤解析
  difficulty  Int      // 1-3
}
```

`options` JSON 结构示例：
```json
{
  "what": ["矩形的面积", "矩形的宽", "矩形的长", "矩形的周长"],
  "method": ["线性方程", "二次方程（因式分解）", "百分比公式", "面积公式"],
  "steps": [
    "设宽为 x，则长为 x+2",
    "建立方程：x(x+2) = 48",
    "展开：x² + 2x - 48 = 0",
    "因式分解：(x+8)(x-6) = 0",
    "解方程：x = 6（舍负）",
    "答：宽为 6"
  ]
}
```

`answer` JSON 结构：
```json
{
  "what": 1,
  "method": 1,
  "steps": [0, 1, 2, 3, 4, 5]
}
```

`explanation` JSON 结构：
```json
{
  "what": "题目要求我们找出矩形的宽，而不是面积（面积是已知条件 48）",
  "method": "已知长和宽有线性关系（长=宽+2），面积是两者乘积，需要用二次方程求解",
  "steps": "完整解题过程见上方步骤"
}
```

## 五、学习引擎集成

### 5.1 进度追踪

每道题的三个步骤分别记录进度：

```prisma
model UserProgress {
  // gameType: "dissector"
  // skill: "what" | "method" | "steps"
  // mathId: 关联题目 id
}
```

### 5.2 SRS 复习

- 三步中任意一步答错，触发该题的复习计划
- 复习时同一道题，但重新经历三步（不透露上次答案）
- 连续三次全对 → 该题"毕业"

### 5.3 题型掌握度

除单题追踪外，还追踪**题型（archetype）维度**的掌握度：

- percentage（百分比题）正确率低 → 推送更多百分比题练习
- 所有 archetype 均毕业 → 进阶到更高难度

## 九、设计约束

1. **不出计算题**：本模块不要求用户做计算，只练读题和规划
2. **不暴露答案**：步骤排序完成前，不显示正确顺序
3. **双语同步**：中英切换后选项文字同步翻译，保持语义一致
4. **难度渐进**：首次使用默认从 difficulty=1 开始，答对后自动推升
