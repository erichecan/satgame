# SAT 数学 · 读题与解题规划模块 PRD（"拆题 / Problem Dissector"）

| 项 | 内容 |
|---|---|
| 模块代号 | 拆题 / Problem Dissector |
| 版本 | v0.1（草案） |
| 状态 | 三阶段可玩 demo 完成（5 题，含中英切换） |
| 定位 | 主产品《SAT R&W 游戏化练习 App》的**数学姊妹模块**；共享引擎部分见主 PRD |
| 一句话定义 | 不练计算，专练**读懂题 + 想清楚怎么解** |

> 注：主 PRD 中曾把"SAT 数学"列为非目标。本模块**有意扩展**该边界，但只覆盖数学的"读题与解题规划"层，不覆盖计算——见 §2。

---

## 1. 背景与问题

- **数学复习 ≠ R&W 复习**：数学是程序性解题，解一道题要几分钟，纯 SRS + 文字小游戏那套套不到核心解题上。
- **关键洞察**：把"分析"从"计算"里拆出来后，分析层就变成几秒钟一次的快速判断——于是它和 R&W 一样**可游戏化、可反复刷**。这是本模块成立的前提。
- **具体痛点**：学生常常只会"转一个弯"的题（所需数据直接给出）；一旦题目**不直接给所需数值、要先求出中间量再用**（转 2–3 个弯），就卡住。这卡的是**读题 + 解题规划**能力，不是算术能力。

---

## 2. 目标与非目标

### 2.1 目标
- 训练数学应用题的三个**分析子技能**：① 读懂在问什么；② 认出该用什么方法/公式；③ 看清要转几个弯并排出步骤。
- 让"弱在多弯"的学生能稳定拆解 2–3 弯的题。

### 2.2 非目标
- **不训练计算本身**（算术、代数运算是另一层能力）。
- 不做全真模拟卷。
- 不要求把题"算到底"——只到**规划出解题路径**为止；最终计算是另一层（见 §13 风险）。

---

## 3. 目标用户与场景
- 同主产品锚点：Grade 9–11 备考 SAT 的学生；当前以 Isabella 为设计锚点（离考约两年，处在打基础阶段）。
- 场景：每天几分钟"拆几道题"；针对错题里"读不懂 / 认错方法 / 漏了一个弯"这类问题做强化。

---

## 4. 设计原则（在主 PRD 七原则之上，本模块特有）
1. **练分析、不练计算**：全程不要求算出数字，只要求读懂、识别、规划。
2. **把"分析"与"计算"显式分层**：这是模块成立的前提，也是和 R&W 玩法对齐的桥。
3. **元技能优先**：三个子技能对四大数学考点域通用，先练这层比逐题型刷题杠杆更高。
4. 其余沿用主 PRD：玩法即练习、反馈讲规则、掌握度记考点、无暗黑模式、AI 当引擎。

---

## 5. 范围：三个子技能 × 数学考点

| 子技能 | 练什么 | 对应 SAT 数学的困难 |
|---|---|---|
| ① 读懂在问什么（The ask） | 快速剥离背景故事，抓住"求什么"和"哪几个数有用" | 应用题被长背景包裹，读得慢、抓错问题 |
| ② 认出用什么方法（The tool） | 一眼识别题型原型/该用的公式 | 不知道这题属于线性/二次/比例/指数…该上哪个工具 |
| ③ 看清转几个弯（The turns） | 判断需要几步、排出"已知→中间量→答案"的路径 | **核心痛点**：题目不直接给所需数值，要先求中间量；多弯题就卡住 |

> 这三个子技能对**代数、高级数学、问题求解与数据分析、几何与三角**四大域通用——是跨题型的元技能。

---

## 6. 核心玩法：拆题（Problem Dissector）

对每道题走**三阶段 + 揭晓**，全程不算数：

- **① 在问什么**：（可计时）读题后点出"这题求什么"（干扰项如"原价""优惠金额"）。错 → 提示"再看一眼最后一句"。
- **② 用什么方法**：不解，识别题型原型/公式；选对后给**识别线索**（例：「每年增长 4%」→ 指数；「每年增加 4」→ 线性）。错 → 提示想识别线索。
- **③ 几个弯**：先猜"转几个弯"，再把**打乱的步骤卡按顺序点出**（已知 → 中间量 → 答案）。直接针对"漏了一个弯"。错 → 提示"先求哪个中间量"。
- **揭晓**：此时才显示算式与答案，并标注"这是一道 N 弯的题"。

**界面要素**：顶部三步进度指示；计时与"利落数"（三步均一次做对算一道利落）；题目卡常驻。

**v1 可扩展**
- ③ 里**掺入"多余干扰步骤"**让用户剔除（更难、更接近真实规划）。
- 揭晓后可选**"现在动手算一遍"**，把规划闭环到计算（见 §13）。

---

## 7. 双语设计
- **默认英文**（读英文原题才是真正要练的能力），右上角一键切**中文**；题干、选项、识别线索、步骤、答案、界面文字全部跟随切换。
- 与解释卡"默认英英、按需中文"一致。
- 用法：先看英文自拆，卡住点中文对照，再切回英文继续。

---

## 8. 学习引擎接入（共享主 PRD §6 + 数学特有）

- **数学版"错题本"**：每次错误按 `(题型原型 archetype, 弯数 turns, 错因)` 记录。错因四类：
  1. 看不懂题　2. 认错方法　3. 漏了一个弯（规划错）　4. 算错
  本模块专攻 **1–3**；"算错"归计算层，本模块不处理。
- **掌握度两个维度**：① 按 archetype（线性/二次/比例/指数/几何…）；② 按**能稳定处理的最大弯数**。系统据此加权出题——"她 1 弯百分比没问题、2 弯就崩"→ 多喂 2 弯题。
- **间隔重现**：到期不是重做原题，而是换一道**同 archetype / 同弯数的新题**（与 R&W 一致）。
- **难度推进轴**：弯数 1→2→3（主难度轴）；archetype 覆盖（广度轴）；干扰步骤（变难手段）。

---

## 9. 内容模型与生成
- 每题富标签（见附录 A）：题干、ask + 干扰、archetype/tool + 干扰 + 识别线索、弯数、**有序步骤**、（可选）干扰步骤、算式解、答案；全部 **EN/ZH 双语**。
- **离线预生成 + 人工审核**，沿用主 PRD §10 的"一次性预生成"决策与理由（SRS 需稳定 ID、内容必须正确）。
- **最需人审的是"步骤分解（弯数与每一步）"**——这一项 LLM 最容易给错。
- 仅"换个说法/动态讲解"为运行时，不产生 Item。

## 11. 成功指标
- 三个子技能各自正确率（ask / tool / turns）。
- **能稳定拆解的最大弯数**（核心成长指标）。
- 各 archetype 的识别准确率。
- 平均拆题用时（速度维度）。
- 错因分布随时间的变化。
- **学习成效（关键）**：复练同 archetype / 同弯数的**新题**正确率随时间上升。

## 13. 风险与开放问题
1. **迁移性（最关键）**：只练"规划不计算"是否真能提分？可能会规划但算错，或脱离计算后规划不扎实。**建议**：部分题在揭晓后加"现在算一遍"闭环；并跟踪"拆题正确率上升"是否带动真实做题正确率。
2. **内容正确性**：步骤分解最易出错，需重点人审。
3. **难度校准**：弯数与 archetype 难度对锚点用户是否合适，需真实试用。
4. **双语默认**：默认英文是否合适，或按题难度自动切中英。
5. **与计算层的边界**：算术错误不在本模块，但体验上要不要顺带练，待定。

## 附录 B：术语
- **archetype**：题型原型（线性/二次/比例/指数/几何/多步…）。
- **turns（弯数）**：解题所需的步骤数，本模块的主难度轴。
- **the ask / the tool / the turns**：三个分析子技能。
- 其余术语（Item、SRS、掌握度、错题本）见主 PRD 附录。
