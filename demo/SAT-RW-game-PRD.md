## 1. 背景与问题

机考版 SAT（Digital SAT）的 Reading & Writing 部分，真题本身就是"一小段文字（25–150 词）+ 一道题"的碎片化结构，天然适合做成"每日小关卡"。

但市面上的 SAT 备考产品存在一个共同问题：**所谓的"游戏"大多只是换了皮的选择题或换了皮的消除**——玩的动作和要练的能力是脱节的。学生玩的时候以为在复习，其实没真正调用考点要的判断，单词和规则成了摆设。

本产品的核心假设：**如果游戏的核心操作本身就要求做出考点要的那个判断，那么"玩"这个动作就等于"练"**。在这个前提下，把碎片化、可坚持、有正反馈的休闲游戏形式，和一套严肃的学习引擎结合起来。

---

## 2. 目标与非目标

### 2.1 产品目标
- 让目标用户形成**每天打一局**的习惯（留存优先）。
- 在不依赖暗黑模式（体力、广告、付费解锁进度）的前提下保持长期黏性。

### 2.2 学习目标
- 覆盖 SAT R&W 四大考点域，且**每个玩法的核心操作都对应一项真实考点判断**。
- 通过间隔重复 + 错题重现，让"不会的词/规则"被有效巩固，而不是刷过即忘。
- 系统能识别用户的**弱项考点**并自动加权练习。

### 2.3 非目标（v1 明确不做）
- 不做全真模拟考 / 计时套题（这是补充工具，不是本产品定位）。
- 不做 SAT 数学，只做 R&W。
- v1 不做实时社交对战、不做 UGC 内容、不做班级/家长后台。
- 不追求"题量最大"，追求"练得对、愿意来"。

## 3. 目标用户与场景

**主要人设（设计锚点）**：Grade 9–11、计划申请美国本科、需要备考 SAT R&W 的学生。时间高度碎片化（课业、课外活动繁重），自制力有限，对传统刷题反感，但对手机小游戏有自然的每日打开习惯。

**典型场景**
1. **每日碎片**：通勤、课间、睡前，5–10 分钟打一局当日任务。
2. **错题巩固**：系统把之前做错的词/规则在隔几天后换语境重新喂回来。
3. **考前冲刺**：临考几周提高频率，按弱项加权强化。

**次要人设（v2 探索）**：英语非母语的成人 / 移民英语学习者——同一套引擎可换词库与考点。

---

## 4. 设计原则（核心，所有玩法与功能都受此约束）

1. **玩法即练习**：游戏的核心操作必须需要做出考点要的判断。绝不做"答对加分"式的换皮选择题。
2. **生成优于识别**：能让用户自己产出答案，就不要只给选项让他认。
3. **反馈讲规则/逻辑，不只讲对错**：答错时要说明*为什么错的选项是陷阱*、*背后是哪条考点规则*。
4. **掌握度记在"考点"上，不记在"题"上**：系统应知道"用户分号不行、过渡词没问题"，据此调整出题。
5. **两种"再来一次"分开设计**：规则型考点用*即时换题重来*（迁移）；记忆型内容用*间隔重现*（隔几天换语境再遇到）。
6. **无暗黑模式**：不设体力条、不插广告中断、不靠付费跳过学习。每日稀缺感来自"一天一局"的仪式，而非人为限制。
7. **AI 当引擎**：用 LLM 做语义裁判、错题讲解、内容生成，而不是把它当噱头。

## 5. 考点范围（SAT R&W Skill Map）

| 考点域 | 真题占比 | 细分规则点（rule） | 对应游戏 |
|---|---|---|---|
| Craft & Structure（技巧与结构） | ~28% | 词在语境中的含义、近义辨析、词义强弱 | Closer / Stronger / Families / Pair Up |
| Information & Ideas（信息与观点） | ~26% | 主旨、推断、找证据、图表 | Read the Green |
| Standard English Conventions（标准英语规范） | ~26% | 句子边界、标点（逗号/分号/冒号/破折号）、主谓一致等 | The Break / Gate Run |
| Expression of Ideas（观点表达） | ~20% | 逻辑衔接（转折/因果/递进/举例/顺序）、组织、修辞综合 | Links / Gate Run |

> 注：阅读理解（Information & Ideas）目前只有 Read the Green 一个玩法覆盖，是 v1 内容缺口，需重点投入。

## 6. 学习引擎（Learning Engine，跨游戏共享）

所有游戏共用同一套引擎；游戏只是引擎之上的不同"皮"与交互。

### 6.1 内容单元（Item）数据模型

```json
{
  "id": "item_00472",
  "type": "vocab | punctuation | transition | reading",
  "skill": "craft_structure | conventions | expression | info_ideas",
  "rule": "clause_boundary | colon_list | contrast | word_intensity | ...",
  "difficulty": 1,                // 1-3，对应机考自适应高低模块
  "payload": { /* 各玩法自定义：词、句子、选项、释义、例句等 */ },
  "distractors": [
    { "text": "...", "trap_type": "near_synonym | out_of_scope | opposite | true_but_irrelevant" }
  ],
  "explanation": "答错时展示的规则讲解（讲为什么，不只讲对错）"
}
```

### 6.2 掌握度模型（Mastery）

两层记录：
- **Item 级**：每个 Item 的 SRS 状态（所在盒子、下次到期时间、连续答对次数、是否已毕业）。
- **Rule 级**：每条考点规则的熟练度分数（0–100），由该规则下所有 Item 的表现聚合。**这是个性化出题与弱项加权的依据。**

### 6.3 间隔重复（SRS = 艾宾浩斯遗忘曲线的实现）

这一节就是"按艾宾浩斯遗忘曲线不断重复"的落地机制。采用简化 Leitner 盒子（实现成本低、效果足够，本质等同于按遗忘曲线拉长复习间隔）：

| 盒子 | 下次出现间隔 |
|---|---|
| Box 0（新/刚错） | 当日 / 最短 |
| Box 1 | 1 天 |
| Box 2 | 3 天 |
| Box 3 | 7 天 |
| Box 4（已毕业） | 14 天+ |

- 答对 → 升一个盒子；答错 → 打回 Box 0。
- **毕业**：在间隔的多次相遇中连续答对 3 次 → 进入长间隔，基本退出日常队列。
- **回退（lapse）**：已毕业的词若日后答错，立即打回 Box 0。

### 6.4 两种"再来一次"

- **即时换题重来**（规则型：标点、连接词）：答错 → 讲规则 → *立刻换一道考同一条规则的不同题* 让其应用（迁移）。
- **间隔重现**（记忆型：词汇）：答错 → 讲含义 → 该词进 Box 0 → 几天后*换一个新语境/新玩法*再遇到。**关键：不重复原题，避免记住题目本身。**

### 6.5 每日组卷算法

每日任务的 Item 组成（默认每日 15–20 题，可配置）：
- ~70% 当日 SRS 到期的复习题
- ~20% 新题
- ~10% 向弱项 rule 加权抽取

不足则用新题补齐；同一个词可在一天内出现在不同玩法中（多语境）。

### 6.6 生词本与按需复习（My Words）

SRS 引擎（§6.3）是"看不见"地在后台跑的；这里补上一个**看得见的生词本**，给用户掌控感和复习入口。

- **自动入册**：用户答错的词、或在解释卡里手动收藏（⭐）的词，自动进入生词本，标注状态（新 / 学习中 / 已掌握）与下次复习日期。
- **复习不另开无聊任务——到期词织回游戏**：到期的词不是丢进枯燥闪卡，而是被重新编排进 Closer / 三消 / 连连看，**换一个新语境**再出现。这是本产品复习机制的核心差异点。
- **按需复习**：用户可随时"复习今日到期词"，拉起一局以生词本词为内容的小关卡。
- **手动加词**：用户可手动添加在别处（学校阅读、文章）遇到的生词；v2 探索"粘贴一句话，自动抽词并生成解释卡"。
- **数据来源**：生词本词条直接复用 §6.1 的 Item / 词库数据，不另建一套内容。

### 6.7 生词解释卡设计（最简，但贴合 SAT）

**设计难点**：最省事的解释是给中文翻译，但 SAT 考的是*词在语境中的含义*与*近义辨析*。纯翻译会把近义词都塌缩成同一个中文、丢掉语体与褒贬，反而误导。

**原则**：用"可控语境 + 褒贬/强弱"来解释，而非只给翻译——因为这正是机考要测的理解方式。解释卡分层呈现，每层都很短：

1. **一句英文简释**（英文先行，养成英英推断习惯）：如 "livid = extremely angry"。
2. **一个语境例句**（看它怎么用，比背定义更懂用法与褒贬）。
3. **褒贬 / 强弱标签**（需要时）：如"比 angry 更强、更正式"。这是 SAT 的辨析点。
4. **近义对比**（需要时）：如 "≠ annoyed（轻得多）"——直接点破 SAT 的陷阱。
5. **默认全英文（仅显示第 1–4 层）**；卡片底部放一个「译成中文」按钮，**点击后才显示中文释义**。中文是按需兜底、不进默认视图，以保住英英推断的训练（中文为预生成文本，点击即显，无需联网或调用 LLM）。

- **出现时机**：在教学时刻展示——答错、揭晓、或消除/配对成功时。
- **"换个说法"按钮**：用户仍不懂时，点击调 LLM 动态换角度 + 给新例句重讲（此为解释，不产生 Item，不影响 SRS；结果可缓存）。
- **生产方式**：第 1–4 层在离线管线中随词库一并预生成并人审（见 §10）；第 5 层翻译同样预生成；"换个说法"为运行时动态。

## 7. 游戏目录（Game Catalog）

统一模板：**考点 / 核心玩法 / 学习闭环 / 错题处理 / 引擎挂钩 / 关键参数 / 状态**。所有玩法均已有可玩 HTML 原型。

### 7.1 词汇类

#### Closer（合成前身 · 语义猜词）
- **考点**：词在语境中的含义（words in context）。
- **玩法**：先给一句*挖空句子* + 词性，用户猜词；每次猜词由 AI 判断与目标词的语义远近（"离靶 N 码"），逐步逼近。
- **学习闭环**：有句子上下文做靶子（不盲猜），练的正是机考真实题型；猜中/揭晓后给释义 + 例句。
- **错题处理**：未猜中进 SRS，间隔重现时换新句。
- **引擎挂钩**：AI 语义裁判（需后端代理）；揭晓即学习时刻。
- **关键参数**：分级提示（先词性 → 首字母+长度 → 揭晓）。
- **状态**：原型完成。

#### Stronger（2048 合成 · 词义强弱）
- **考点**：近义词的**强弱顺序与精确度**（市面极少练）。
- **玩法**：4×4 滑动；两个相同词合成更强一级（annoyed→irritated→angry→furious→livid）。
- **学习闭环**：通过合成动作内化词义梯度；侧栏梯子图实时显示已达层级。
- **引擎挂钩**：每条 word ladder 是一组同义梯度 Item；到顶 = 掌握该梯度。
- **关键参数**：当前词链偏日常，**待办：替换为更贴近 SAT 难度的词链**。
- **状态**：原型完成。

#### Families（三消 · 归类 + 词义闸门）
- **考点**：近义辨析、归类、精确含义。
- **玩法**：交换相邻词凑齐 3 个同类（情绪家族）；**消除前必须答对"这个词什么意思"三选一才能清除并得分**（一次答对得双倍）。颜色仅辅助扫视。
- **学习闭环**：色彩保证可玩性，但"得分+消除"这道闸门卡死在词义上——单词不再是摆设。连锁消除免答、直接送分以保持手感。
- **错题处理**：答错标灰继续选直到答对；答错的词记入 SRS（待接）。
- **关键参数**：当前干扰项取自其他家族（较易），可选改为同家族不同强弱（更练精确度）。
- **状态**：原型完成（已含词义闸门）。

#### Pair Up（连连看 · 配对消除）
- **考点**：词义记忆、反义关系。
- **玩法**：点词 → 点其匹配项，全清即赢，记失误数。两模式：**词 → 释义** / **反义词**。
- **学习闭环**：最直接的词义/反义记忆。
- **关键参数**：每屏 6 对（12 张）；可加 连连看 的"路径可连"约束增加挑战（v2）。
- **状态**：原型完成。

### 7.2 规则类（标点 / 连接词）

> 设计定位：同一批规则有两种表达——**"学习模式"（带脚手架，给新规则用）** 与 **"速度模式"（限时，给复习用）**，对应引擎的 新题 vs 复习 区分。

#### The Break（标点拖拽 · 学习模式）
- **考点**：句子边界与标点（逗号粘连、分号/冒号/破折号、插入成分）。
- **玩法**：句子留空，拖或点正确标点进缺口；答错即时讲规则并换同规则新句。
- **学习闭环**：放置前必须判断"两边是否各自成句"——底层技能。
- **状态**：原型完成（含拖拽 + 点击两种交互）。

#### Links（连接词 · 学习模式）
- **考点**：逻辑衔接（转折/因果/递进/举例/顺序）。
- **玩法**：**两步**——先判断两句的*关系*，再选具体连接词。
- **学习闭环**：把判断重心放在"识别逻辑关系"这一步，而非背连接词列表。
- **状态**：原型完成。

#### Gate Run（跑酷双门 · 速度模式）
- **考点**：标点 + 连接词（规则有限、适合刷速度）。
- **玩法**：计时条缩短，撞门前选对的那扇（","vs";"、However vs Therefore）；选对加速，错/超时扣心，三心结束。
- **学习闭环**：在压力下快速调用已学规则，模拟机考节奏。
- **关键参数**：起步 4.2s → 最快 1.8s（随分数缩短），3 颗心。**待办：节奏需对真实用户校准。**
- **状态**：原型完成。

### 7.3 阅读类

#### Read the Green（微阅读 · 答题 + 找证据）
- **考点**：主旨、推断、找证据（Information & Ideas）。
- **玩法**：一小段（3–4 句）+ 一道四选一；**答对后还需在原文里点出"能证明该答案的那一句"**。
- **学习闭环**："找证据"是 SAT 奖励的元认知技能；答错时按陷阱类型（太宽泛/没提到/相反/对但不相关）讲解。
- **关键参数**：当前仅 3 篇，**v1 内容缺口最大，需重点扩充**。
- **状态**：原型完成。

## 8. 每日循环与留存

- **每日任务**：一组按 §6.5 组卷的题，跨多个游戏分布；完成即"今日打卡"。
- **连胜（streak）**：连续天数；断签提醒。
- **可分享结果**：仿 Wordle 的不剧透色块/成绩卡，可发给同学比。
- **结算页**：今日正确率、巩固了哪些词/规则、连胜、最弱考点提示。
- **提醒**：每日一条本地推送（时间可设）。
- **明确不做**：体力条、强制广告、付费跳过进度。

## 9. 信息架构与界面（IA & Screens）

1. **主页**：今日任务入口 + 连胜 + 各游戏单独入口（自由练）+ 当前最弱考点卡片。
2. **游戏页**：各玩法独立界面（见 §7 原型）。
3. **结算页**：见 §8。
4. **进度页**：四大考点域的掌握度可视化 + 弱项 rule 列表 + 已毕业/待复习词数。
5. **生词本（My Words）**：按状态（新/学习中/已掌握）列出词条 + 下次复习日期；支持"复习今日到期"、手动加词、查看解释卡（见 §6.6）。
6. **设置**：提醒时间、难度、玩法开关、词库选择、**解释卡默认英英（每张卡可点「译成中文」）；可选「总是展开中文翻译」**。

## 10. 内容策略与生成

### 10.1 内容类型

| 类型 | 内容 |
|---|---|
| 词库（canonical） | 词、词性、英文简释、≥3 个语境例句、同义/反义、强弱词梯、褒贬/语体标签、中文释义、难度 |
| 题库（derived） | Closer 挖空句、标点题、连接词对、微阅读短文+题；每题带 rule / difficulty / 干扰项 + 陷阱类型 |
| 解释卡 | §6.7 的分层解释（简释/例句/标签/对比/翻译） |
| 运行时判定 | Closer 的语义远近评分、"换个说法"的动态解释（不入库，可缓存） |

### 10.2 核心决策：一次性预生成，而非运行时动态生成

**结论：所有题目与词库内容一律离线批量生成 + 人工审核后入库，静态下发。运行时不现生成题目。**

| 维度 | 预生成（采用） | 运行时动态生成（否决） |
|---|---|---|
| **SRS 兼容**（决定性） | Item 有稳定 ID，可追踪历史、按曲线复现 | 每次新内容无法追踪掌握度，**整套复习引擎失效** |
| 正确性 | 可人审，杜绝错误"正确答案" | 可能给出错误/有歧义的题，等于教错 |
| 成本 / 延迟 | 服务时近乎零成本、即时 | 每局调用 LLM，贵且慢 |
| 可复现 / 可调试 | 是 | 否 |

**仅两处保留动态**（均不产生 Item、不影响 SRS、结果可缓存）：① Closer 的语义远近裁判（必须实时）；② 解释卡的"换个说法"。

### 10.3 离线生成管线（LLM 当"内容工厂" + 人审）

1. **种子词表**：从成熟来源/词频表整理 SAT R&W 高频词（words-in-context 取向，非孤立背词）。
2. **LLM 富化**：为每词生成词性、英文简释、≥3 个不同语境例句、同义/反义、所属强弱词梯、褒贬/语体标签、中文释义、难度标签。
3. **LLM 派生题目**：据词库生成各玩法 Item——Closer 挖空句、三消家族归属、连连看配对、标点/连接词题，并为每个干扰项标注陷阱类型。
4. **人工审核（强制）**：接受/编辑/拒绝。**词梯排序与近义分组最易出错、最需重点审**，因为那正是 SAT 考的细微差别；其次是干扰项是否唯一排除、是否贴合机考风格。
5. **入库**：写入带版本的内容库，打全 rule / skill / difficulty 标签。
6. **持续扩充**：后台离线任务逐步扩大词库与题量；同一词预生成**一个语境例句池**，供 SRS 重现时换新句。

### 10.4 QA 与对齐
- 每条 Item 校验：唯一正确答案、干扰项合理且可排除、与 rule/考点对齐、与机考 Digital SAT 的格式与难度分布一致。
- 词梯、近义组额外做一致性抽检。

### 10.5 规模估算（v1 起步）
- 核心词 ≥300；标点题 ≥80（覆盖 6–8 条规则）；连接词题 ≥80；微阅读 ≥60 篇。
- 微阅读题量缺口最大，是 v1 内容重点投入项。

## 11. 技术架构

- **客户端**：v1 用 Web / PWA（现有原型即 HTML，可快速迭代）；v2 视情况上 React Native。
- **后端服务**：
  - 内容服务（Item 库 + 词库）
  - 进度服务（用户 SRS 状态、掌握度、连胜）
  - **LLM 代理服务**：封装对 Anthropic API 的调用，**绝不在客户端暴露 key**。
- **LLM 用途**：① Closer 的语义远近裁判（运行时）；② 解释卡的"换个说法"动态重讲（运行时）；③ 离线内容生成管线（见 §10.3）。其中 ①② 走后端代理 + 缓存，③ 为离线批处理。
- **成本/性能**：语义裁判结果按 (目标词, 猜测词) **缓存**；加**限流**；裁判输出走严格 JSON 解析 + 失败兜底（提示重试，不崩游戏）。
- **离线**：词库与题库可预下载，纯本地玩法（Stronger/Families/Pair Up/The Break/Links/Read the Green）应能离线运行；仅 Closer 的实时裁判需联网（可做离线降级：预生成的近义表）。
- **数据埋点**：每题的答对/错、用时、重试次数、放弃；每日完成率、连胜、各游戏时长。

> 现状说明：当前原型为演示，Closer 直接调用了 in-artifact API；生产环境必须改为后端代理 + 缓存 + 限流。

## 12. 成功指标（KPIs）

- **北极星指标**：周活跃中"每周打卡 ≥4 天"的用户占比。
- **留存**：D1 / D7 / D30 留存；平均连胜长度。
- **参与**：每日完成率、人均每日局数、单局完成率。
- **学习成效**（最关键、最难，但必须测）：
  - 复习题正确率随时间上升（掌握度增长曲线）。
  - 已毕业词/规则数量。
  - 弱项考点的正确率收敛速度。

## 14. 风险与开放问题

1. **学习成效难证明**：游戏化容易上瘾但未必提分，需尽早设计"复习正确率上升"等成效指标并真实跟踪。
2. **AI 裁判一致性/成本**：Closer 同一对词多次评分是否稳定？缓存与限流策略需验证。
3. **手机端可玩性**：三消 5 列英文词是否过挤；拖拽在移动端是否顺畅（已做点击兜底）。
4. **趣味与学习的平衡点**：Families 的词义闸门频率（每次消除 vs 仅大消除/连击时触发）需用真实用户校准。
5. **内容缺口**：阅读理解（Read the Green）玩法单一、题量最少，是 v1 最大投入项。
6. **词链/题库的 SAT 对齐度**：现有原型词偏日常，需替换为 SAT 难度并人审。
