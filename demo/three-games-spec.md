# 三个游戏：玩法拆解与实现规格（Connections / Wordle / Strands）

> 目标：把 NYT 这三个游戏的玩法吃透 + 给出可直接开发的实现要点 + 标注对 SAT 的适配与工作量。
> 说明：NYT 页面有登录墙抓不到，规则以你提供的官方"How to Play"截图为准。**Connections 已有可玩原型（sat-connections.html）。**

---

## 0. 总览与建议

| 游戏 | 一句话玩法 | 实现难度 | 对 SAT 契合度 | 建议顺序 |
|---|---|---|---|---|
| Connections | 16 个词分成 4 组 | ★（最易，已有原型） | ★★★（最佳：练辨析） | 已有 |
| Wordle | 6 次猜出 5 字母词 | ★★（成熟算法） | ★（最弱：练拼写） | 第 2 个 |
| Strands | 在字母网格里找出铺满全盘的主题词 | ★★★★（出题最难） | ★★（主题=语义类别） | 第 3 个 |

核心结论：Connections 是 SAT 辨析的最佳载体；Wordle 好做但学习价值偏拼写；Strands 玩法独特、契合尚可，但**内容生产是个硬骨头**。

---

## 1. Connections（最易，已有原型）

### 1.1 玩法规则
- 16 个词，分成 4 组、每组 4 个，按"共同点"归类。
- 选中 4 个 → Submit 检查。对 → 该组锁定并显示颜色；错 → 记一次失误。
- **只能错 4 次**，错满即结束（揭晓答案）。
- 分类**比"5 个字母""动词"这种更具体**；**每个谜题恰好一个解**。
- 难度四档颜色，随解出逐步揭示：黄(送分) → 绿 → 蓝 → 紫(最阴险)。
- 官方示例：`FISH: Bass, Flounder, Salmon, Trout`；`FIRE ___: Ant, Drill, Island, Opal`。

### 1.2 关键细节
- **陷阱是灵魂**：故意放"看着属于多组"的词，难度来自歧义而非生僻。
- **One away**：提交的 4 个里有 3 个属于同一真实组时，提示"就差一个"。
- 紫组常是"词义最细 / 带文字游戏"的一组。

### 1.3 实现
- 数据模型：
```json
{ "groups": [ { "tier":0, "theme":"Praise", "members":["laud","extol","commend","applaud"] }, ... ] }
```
- 判定逻辑：选中 4 个 → 统计它们各属哪组 → 若全属同一组则解出；否则失误，并计算与某单组的最大重叠数，==3 即"one away"。
- 失误 4 次 → 揭晓剩余组；全解出 → 胜。
- UI：4×4 网格、最多选 4、Submit/Shuffle/Deselect、解出组以颜色条堆叠在上方。

### 1.4 SAT 改造（= 我们已做的 Clusters）
- 4 组按**精确含义 / 褒贬 / 语体**划分，组间放**近义陷阱**——一个只模糊知道词义的学生会按"大概褒还是贬"瞎归，而这正是 SAT 考的辨析。
- **定位：复习/巩固向**（适合已学过的词；全新词无法归类）。

## 2. Wordle

### 2.1 玩法规则
- 猜一个隐藏的 **5 字母词**，最多 **6 次**。
- 每次猜必须是**合法 5 字母词**。
- 每格着色反馈：
  - **绿**：字母正确且位置正确。
  - **黄**：字母在词中但位置不对。
  - **灰**：字母不在词中。
- 6 次内全绿即胜；用完 6 次未中即负。每日一题，午夜更新。
- 键盘按键随累计反馈染色。

### 2.2 关键细节（必须处理对）
- **重复字母**是最容易写错的地方，要用**两遍着色法**：
```
score(guess, target):
  result = [gray]*5
  count = 每个字母在 target 中的出现次数
  # 第一遍：标绿，并扣减对应字母计数
  for i in 0..4:
    if guess[i] == target[i]:
      result[i] = green; count[guess[i]] -= 1
  # 第二遍：标黄（仅当该字母还有剩余计数）
  for i in 0..4:
    if result[i] == green: continue
    if count[guess[i]] > 0:
      result[i] = yellow; count[guess[i]] -= 1
  return result
```
  这样保证：guess 里某字母数量多于 target 时，多出来的标灰而非全黄。
- **两套词表**：`solutions`（答案池，较小、经筛选）和 `allowed`（可接受的猜测词，更大）。猜测词必须在 `allowed` 里，否则拒绝（"不在词库中"，该行抖动）。
- **键盘染色优先级**：绿 > 黄 > 灰（一个键显示它达到过的最好状态，不被后续降级）。

### 2.3 实现
- 数据模型：`{ target, solutions[], allowed:Set }`、`guesses[]`、`current`、`keyState:{letter->status}`。
- 输入：物理键盘 + 屏幕键盘；Enter 提交、Backspace 删除。
- UI：6×5 网格；提交时翻牌动画；非法词行抖动；键盘染色。
- 边界：非法/长度不足拒绝；末行命中即胜；可选 Hard Mode（已揭示的线索必须沿用）。

### 2.4 SAT 改造与适配（**最弱契合，如实说**）
- Wordle 本质练**拼写 + 词形回忆**，不练词义。
- **5 字母硬约束**会砍掉大量 SAT 词（多数不是 5 字母）——SAT-Wordle 的词池很有限。
- 想加学习价值的两条路：① 解出后弹出该词的释义/例句（顺带学义）；② **"释义当线索 → 猜出对应的词"**变体（回忆 + 拼写），但仍受 5 字母限制。
- 结论：作为**拼写/熟悉度的轻量层**可以，别指望它扛词义训练。

## 3. Strands（最难）

### 3.1 玩法规则
- 字母网格（NYT 正式版 **6 列 × 8 行 = 48 格**；你的截图是简化示意）。
- **找出所有"主题词"**：所有主题词**无重叠地铺满整个网格**（每个字母恰好属于一个主题词）。
- **主题词找到后高亮蓝色**。
- **Spangram（横贯词）**：一个特殊词（可能多词），**横贯棋盘、触及相对的两条边**（上↔下 或 左↔右），概括整个主题，找到后高亮**黄色**。
- 成词方式：相邻字母连成路径（**含对角线，8 向**，像 Boggle）；**拖动**划过，或**点击**字母、**双击最后一个字母**提交。
- **没有失误上限**：不会因猜错而输；目标是把全盘找完。
- **提示**：找到**非主题词**（合法英文词）可换提示——**每找 3 个非主题词得 1 次提示**；提示高亮某个主题词所在的格子；若已有激活的提示，下一次提示会给出该词的**字母连接顺序**。

### 3.2 关键细节与三大难点
1. **出题 = 约束满足问题（最难）**：要找到一组主题词 + spangram，使它们的路径**恰好铺满整个网格、互不重叠**，且 spangram 触及相对两边。这几乎必须**离线用生成器/求解器产出，或人工排盘**——不可能运行时现生成。这是三个游戏里最大的内容工程。
2. **路径输入 UX**：拖动连格（pointer 划过）或点选 + 双击提交；移动端拖动易出错，建议**点选成词**为主、拖动为辅（与标点拖拽那关同思路）。
3. **需要一个英文词典**：用于识别"非主题词"以发提示（合法但非主题的词）。需内置词表（Set/Trie）。

### 3.3 实现
- 数据模型（每个谜题，离线产好）：
```json
{
  "theme": "Praise",
  "rows": 8, "cols": 6,
  "grid": [["L","A","U","D",...], ...],
  "themeWords": [ { "word":"EXTOL", "path":[[0,0],[0,1],[1,1],...] }, ... ],
  "spangram": { "word":"PRAISE", "path":[...] }
}
```
  另需全局英文词典用于非主题词判定。
- 路径校验（运行时）：给定选中的格子序列 →
  1. 相邻性：相邻两格必须 8 向相邻；
  2. 不重用：路径内无重复格；
  3. 成词：拼出的字符串若 == 某未找到的主题词 → 锁定蓝；== spangram → 锁定黄；若是词典里的合法词但非主题 → 计入非主题词计数（攒提示）；否则拒绝。
- 提示系统：维护已找到的不同非主题词数；每满 3 → 高亮一个未找到主题词的格子；记录"激活提示"，下次提示给出其字母顺序。
- 胜利：全部主题词 + spangram 找齐（满盘蓝 + 黄）。

### 3.4 SAT 改造与适配
- **主题 = 语义类别**，**spangram = 类别名**。例：主题"praise"，spangram = `PRAISE`，主题词 = `LAUD / EXTOL / COMMEND / HAIL ...`。
- 训练价值：识别"属于某含义类别"的词，spangram 点破类别——契合不错。
- **但**：铺满网格的约束意味着主题词的**长度组合必须能拼满盘**，用特定 SAT 词去满足这个约束**非常难**——这是 SAT-Strands 内容生产的核心障碍。可考虑放宽盘面尺寸或允许更多备选词来缓解。

## 4. 三个游戏共用的底座（沿用主 PRD）
- **每日一题 + 连胜 + 可分享结果**（主 PRD §8）。
- **内容离线预生成 + 人审、一次性而非动态**（主 PRD §10）；Strands 的"铺盘求解"是其中最重的一环。
- **接 SRS / 掌握度**：Connections 与 Strands 的"词"可计入生词本与按词义类别的掌握度；Wordle 的词计入拼写/熟悉度。

---

## 5. 下一步建议
1. 先把 **Connections** 原型补成正式版（每日题、连胜、题库）。
2. 做 **Wordle**：算法成熟，1–2 天可出可玩版；先定 SAT 5 字母词池。
3. **Strands** 分两步：先做"给定已排好的盘面 → 能玩"的运行时；**再单独立项做"自动铺盘生成器"**（最难，建议先人工排几盘验证玩法）。
