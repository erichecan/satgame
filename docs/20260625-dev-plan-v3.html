Part I · 为什么这么做
产品与 SAT 备考策略结论
本次讨论沉淀的判断 —— 产品方向 + SAT 应试策略

S1产品评估结论

设计的教育内核是对的，但铺得太开、且高估了代码、低估了内容。

  • 教育内核站得住 — 考点维度掌握度 + 迁移练习 + evidence-based 方向，是真懂教育学的决策，最值钱。
  • 风险一 · 范围过大 — 9 个游戏给一个人用，ROI 失衡；真正决定提分的就 4-5 个。
  • 风险二 · 内容是瓶颈（不是代码） — 每个游戏要喂进精审内容才有价值，真正的工作量在这。
  • 风险三 · 好玩 ≠ 提分 — 2048 / 三消娱乐性 > 教学有效性，易"玩得开心却没练到点子上"。
核心建议 · 垂直切片：先做 5 个核心游戏 + streak，让 Isabella 真的每天用起来——验证习惯 > 堆功能。

S2SAT 题目如何融入：记忆类 vs 题型类

数字 SAT 几乎不考孤立单词与拼写，考的是 Words in Context（语境中的词义）。你对 Wordle 的判断是对的。

游戏SAT 相关度定位
Five（Wordle）低 · SAT 不考拼写记忆玩具，附属
Clusters / 词汇辨析中词汇巩固
Closer（语境填词）高 · 几乎 1:1 对应真实题型核心提分
Gate Run（标点/过渡词）高 · 直接对应语法题核心提分
Read the Green（阅读证据）高 · 占比最大题型核心提分
把游戏分两类：记忆类（Wordle，锦上添花）vs 题型类（Closer / Gate Run / Reading，核心提分）。资源压在题型类上。

S3数学：除读题外的练习形式

读题（Dissector）练的是"解题规划"，只是一个维度。其它高价值形式：

  • 陷阱识别 — SAT 数学错误选项是精心设计的坑（算到一半的中间值、问长答宽）。专练识别，最贴 SAT。
  • 同型快刷 / 模式识别 — 同 archetype 连刷 5 题，练到"看一眼就知道套路"。
  • 估算 / 排除 — 全程可用计算器，高分靠快速估算 + 选项排除省时间。
  • 概念配对 — 补概念盲点（连连看式，用数学概念）。
关键事实：数字 SAT 数学是自适应的——第一模块表现决定第二模块难度。"稳定不犯低级错"比"会做难题"更重要 → 陷阱识别价值最高。

S4高频词和数学题能自动生成吗

能，但词汇和数学的风险完全不同，必须分开处理。

  • 词汇 ✓ 可以且应该 — 高频词表是现成的（不用"生成"）；配套内容（释义/例句/同反义/分组）用 LLM 批量生成 + 人工抽审。
  • 数学 ⚠ 别让 LLM 裸生成 — 答案可能算错、可能多个选项都对、难度标定不准。
  • 数学安全做法 — 参数化模板（题骨架 + 随机数 + 代码算答案，答案永远对，可无限出题）；或 LLM 出题 + 代码验证唯一性。
一句话：词汇靠 LLM + 人审，数学靠模板 + 代码验证。

S5SAT 官方目的与规律

College Board 官方定位：SAT 测"大学与职业准备度"（college & career readiness），考迁移 / 分析 / 推理，不是知识量。

  • R&W — 跨学科阅读 + 语言运用，强调 evidence-based（基于证据）；2024 起全面数字化、自适应。
  • Math — 代数 / 高级数学入门 / 数据分析 / 几何三角，强调"在真实情境中应用"。
  • 总原则 — 考的是 reasoning，不是偏题怪题；题目有规律、可预测。
对设计的启示：官方既然考"迁移能力"，那么"同一规则换情境反复练" > "背题刷题"——这正印证了你"掌握度记在考点维度 + 即时迁移 / 间隔重现"的设计。方向对，要坚持。

S6如何取得 1540

1540 / 1600 ≈ 99 百分位，全球前 1%。

  • 分数构成 — 典型 R&W ~760 + Math ~780；数学满分容错极小（错 1-2 题掉 20-40 分）。
  • 靠什么 — 消灭系统性错误，不是多做题；错题归因比刷新题重要 10 倍。
  • 需要 — 词汇语境扎实 + 语法零失误 + 数学无盲点且不粗心，三者同时满足。
  • Grade 9 优势 — 现在起步时间充裕；但 1540 是 1-2 年的积累，不是一个暑假能冲出来的。
唯一标准尺：官方 Bluebook app 的全真模考。任何第三方题都只是模拟，分数不算数。

S7如何练 · 暑期计划

Grade 9 暑假目标不是冲 1540，是打地基 + 养成每日习惯。

  • 第一步 · 诊断 — 先做一套 Bluebook 全真模考，拿到基线分。不知道起点，后面都是瞎练。
  • 每日 30-60 分钟 — 词汇（game）/ 语法（Gate Run）/ 数学（Dissector）/ 阅读（Read the Green）。可持续才是关键。
  • 每周一次 — 一个模块小模考 + 错题复盘归因（这步最值钱，别跳）。
  • game 的定位 — 每日习惯养成器 + 日常练习场；真题（Bluebook）留作每周/每月检验。两者分工。

02开发的真实性质：移植，而非重写

每个保留的 demo 已是单文件、可玩、视觉成品级。MVP = 对每个游戏做这 5 件事：

① 移植
HTML/JS → React 组件
Next.js + TypeScript
② 数据外迁
硬编码题目 → 数据库种子
题目即内容资产
③ 接进度
统一 /api/progress
SRS + XP + streak + 徽章
④ Closer 后端化
前端直调 → 后端代理
+ 24h 缓存，安全
⑤ 壳
首页 / 导航 / 生词本
成就页
注意：Closer 的 demo 直接在前端调 api.anthropic.com（无 key、不安全），生产必须改为 /api/semantic-distance 后端代理；无 key 时降级为预设近义词表。

04Gamification（单用户改造）

单用户下排行榜无意义，改造为以下三机制：

  • Streak 连续天数 — Duolingo 命脉，单用户最强留存。第一个做。
  • XP 积分 + 等级 — 每答对 / 每「毕业」一个考点给分，进步可量化。
  • 徽章（里程碑） — 掌握 50 词 / 连续 7 天 / 攻克分号规则 / 数学读题满分。
表 UserStats totalXp · level · currentStreak · longestStreak 表 Achievement code · progress · target · unlockedAt 挂在 /api/progress 顺手结算，无新架构

06风险点

  • 视觉统一 — 两代 demo 风格不一，需定一套统一设计 token（建议统一到现代 Space Grotesk 主题）。
  • 内容扩充 — demo 每游戏仅 3-5 题，够验证体验、不够长期用。后续：词汇/阅读/语法走「LLM 生成 + 人审」，数学走「参数化模板 + 代码验证」。
  • Closer 依赖 Claude API — 必须后端代理，配 ANTHROPIC_API_KEY，无 key 降级。
  • 单用户 — 无注册登录，进度无 userId（未来多用户再加）。
